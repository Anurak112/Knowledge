# API Documentation Standards / มาตรฐานการเขียนเอกสาร API

## Guidelines
1. **OpenAPI (Swagger)**: All REST APIs must have an OpenAPI 3.0 spec.
2. **Examples**: Every endpoint must have request/response examples.
3. **Authentication**: Clearly document auth headers (Bearer token, API Key).
4. **Errors**: Document all error codes (400, 401, 403, 404, 500).

## Tools
- Swagger UI
- Postman Collections
