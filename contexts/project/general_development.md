# General Development Context / Context สำหรับการพัฒนาซอฟต์แวร์ทั่วไป

## Goals
- Write clean, maintainable code.
- Follow SOLID principles.
- Ensure high test coverage.

## Workflow
1. **Plan**: Design before coding.
2. **Code**: Implement small, testable units.
3. **Review**: Peer review and automated checks.
4. **Test**: Run unit and integration tests.
5. **Deploy**: Automated deployment pipeline.

## Standards
- **Naming**: camelCase for variables, PascalCase for classes.
- **Commits**: Conventional Commits format.
- **Documentation**: JSDoc/Docstrings for public APIs.
